Database#

Milvus向量数据库#

Milvus相关的封装在common/utils/milvus_utils.py中，继承了pymilvus中的Collection类。

  1. ConfigMilvusWrapper支持用yaml文件配置表的字段，配置文件在im_client/configs/vecdb中, eg: agent_contact.yaml。

    当两个Milvus Collection的name和using字段相同时，会连接到同一张表。可以通过name_modify和connection_modify参数修改name和using.

  2. 插入/删除数据 
    example = ConfigMilvusWrapper('example.yaml')
# example: key | name | desc
example['A'] = {'name': 'Alice', 'desc': 'Alice is ...'}

example['A'] = <ClassName>（'name'= 'Alice', 'desc'= 'Alice is ...')

example.upsert_data({'key':'A', 'name': 'Alice', 'desc' 'Alice is ...'})

example.remove(<key>)# 删除主键为< key>的entity

    上面三种都会向向量数据库插入一条 key | name | desc 的数据。如果设置desc的auto_vectorize为True, 会同时插入desc_vec数据，为向量化后的desc_vec。

  3. 建立索引

    在进行ANN查询前需要建立索引。如果设置auto_create_index==True并且配置了一条index规则，会自动设置。

    example.index_via_config(<name>) # name缺省时使用设置的第一条规则
  4. 查询

    在进行ANN查询前需要建立索引。如果设置auto_create_index==True并且配置了一条index规则，会自动设置。

    # input: vec, list[vec], str, list[str](auto_vectorize=True)
 example.search_via_config(input, name=<name>)
# output: [{f: v}], [[{f: v}]] # 只输出output_fields中指定的field
  5. 建立索引

    其他: __getitem__(key), __contains__(key), keys(), items(), __len__()

Sqlite#

Sqlite相关的工具包含AutoStoredDict和AutoStoredSet,位于common/utils/database_utils.py

  1. AutoStoredDict继承自SqliteDict，支持与dict类似的数据库操作：

    example = AutoStoredDict('test.db', tablename='test')
example['A'] = 'Alice'
example == {'A':'Alice'} # True

  2. AutoStoredSet在表中建立一个key='__set'

    example = AutoStoredDict('test.db', tablename='set_test')
example.add('a')
example == set('a') # True
